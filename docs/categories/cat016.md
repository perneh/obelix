# Category 016 – INCS Configuration Reports

**Edition:** 1.0 (Eurocontrol ASTERIX Part 30)  
**Purpose:** Configuration data for INCS systems — transmitter/receiver pairs and reference points.

## Overview

Category 016 reports **INCS system configuration**. When used with **Category 015**, the pair identifier in both categories allows tracing a target report to the Tx/Rx pair that contributed to it.

## Implemented data items

| Item | FRN | Obelix field |
|------|-----|--------------|
| I016/010 Data Source Identifier | 1 | `data_source` |
| I016/015 Service Identification | 2 | `service_id` (optional) |
| I016/000 Message Type | 3 | `message_type` |
| I016/140 Time Of Day | 4 | `time_of_day` |
| I016/200 Reporting Period | 5 | `reporting_period_s` (optional) |
| I016/300 Pair Identification | 6 | `pair` (one pair: PID, TID, RID) |
| I016/400 System Reference Point | 7 | `reference_point` lat/lon |
| I016/405 Height of Reference Point | 8 | `reference_point.height_m` |

## Message types

| Value | Meaning |
|-------|---------|
| 1 | System configuration |
| 2 | Transmitter / receiver configuration |

## Linking to Cat 015

Set the same **Pair ID** in:

- Cat 016 step: `pair.pair_id`
- Cat 015 step: `measurement_id.pair_id`

## Shared configuration

Load **INCS System Configuration** from `configurations/cat016/incs-system-config.json`.

## Wireshark

```text
asterix.category == 16 && udp.port == 8600
```

## Limitations

Obelix encodes one pair per message and does not include repetitive transmitter/receiver property items (I016/410, I016/420). Extend `cat016.py` for full component metadata if needed.
