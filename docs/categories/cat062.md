# Category 062 – System Track Data

**Edition:** 1.21 (Eurocontrol ASTERIX Part 9)  
**Reference:** [EUROCONTROL-SPEC-0149-9](https://www.eurocontrol.int/publication/cat062-eurocontrol-specification-surveillance-data-exchange-asterix-part-9-category-062)

**Purpose:** Processed system track messages from an SDPS (Surveillance Data Processing System).

## Overview

Category 062 carries **fused system tracks** — the output of a surveillance data processing system after correlation, smoothing, and height/velocity estimation. This is the primary category for simulating ARTAS/SDPS track output and testing track consumers.

Obelix follows the Eurocontrol default UAP mapping and encoding rules for the implemented items.

## Mandatory items

These are always included in encoded messages:

| Item | FRN | Obelix field |
|------|-----|--------------|
| I062/010 Data Source Identifier | 1 | `data_source` |
| I062/070 Time Of Track Information | 4 | `time_of_track` |
| I062/040 Track Number | 12 | `track_number` |
| I062/080 Track Status | 13 | `track_status` |

**Time Of Track Information** uses seconds since last midnight UTC; LSB = 1/128 s.

## Optional items

| Item | FRN | Obelix field | Include when |
|------|-----|--------------|--------------|
| I062/015 Service Identification | 3 | `service_id` | Value > 0 |
| I062/105 WGS-84 Position | 5 | `wgs84` | Position format = WGS-84 |
| I062/100 Cartesian Position | 6 | `cartesian` | Position format = Cartesian |
| I062/185 Track Velocity | 7 | `velocity` | Include velocity = Yes |
| I062/060 Mode 3/A | 9 | `mode3a.code` | Code > 0 |
| I062/136 Flight Level | 17 | `flight_level` | Value ≠ 0 |
| I062/130 Geometric Altitude | 18 | `geometric_altitude_ft` | Include geometric altitude = Yes |

### Position format

Choose one position representation:

| Value | Format | Item |
|-------|--------|------|
| Cartesian | Local X/Y in metres | I062/100, LSB = 0.5 m |
| WGS-84 | Latitude / longitude in degrees | I062/105, LSB = 180/2²⁵ ° |

Only the selected format is encoded.

### Track status (I062/080)

First octet only (FX=0). Controls in the UI map to:

| Bit field | UI control |
|-----------|------------|
| MON | Monosensor vs multisensor track |
| SPI | Special position identification |
| MRH | Barometric vs geometric as most reliable height |
| SRC | Altitude source (3 bits) |
| CNF | Confirmed vs tentative track |

### Mode 3/A (I062/060)

Enter code in **octal** (0–7777). Set code to `0` to omit the item. Status flags (validated, garbled, changed) are encoded in the item header bits.

## Encoding resolutions

| Item | LSB |
|------|-----|
| I062/070 Time | 1/128 s |
| I062/100 Cartesian | 0.5 m |
| I062/105 WGS-84 | 180/2²⁵ degrees |
| I062/185 Velocity | 0.25 m/s |
| I062/130 Geometric altitude | 6.25 ft |
| I062/136 Flight level | 0.25 FL |

## Shared configuration

Load **System Track (WGS-84)** from the library (`configurations/cat062/system-track-wgs84.json`) — Stockholm area track with velocity, Mode 3/A 7000, FL350, and geometric altitude.

## Scenario route animation

In the **Scenario Builder**, enable **Animate route** on a Cat 062 step to move the track between waypoints instead of sending a static position. Set the end latitude/longitude (or Cartesian X/Y), **Ticks** (number of updates), and **Interval (ms)**. Obelix interpolates position on each tick and can advance **Time Of Track** and derive **velocity** automatically.

## Limitations

Obelix implements a **practical subset** of Cat 062 for testing. Not encoded today:

- Aircraft identification, target address, vehicle fleet ID
- Track quality, accuracy, and covariance items
- Sensor status, MDS data, ACAS/resolution advisories
- Multi-extent track status beyond the first octet

For production SDPS integration, compare against the full Eurocontrol UAP and extend `cat062.py` as needed.
