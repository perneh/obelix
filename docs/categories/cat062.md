# Category 062 – System Track Data

**Edition:** 1.21 (Eurocontrol ASTERIX Part 9)  
**Reference:** [EUROCONTROL-SPEC-0149-9](https://www.eurocontrol.int/publication/cat062-eurocontrol-specification-surveillance-data-exchange-asterix-part-9-category-062)

**Purpose:** Processed system track messages from an SDPS (Surveillance Data Processing System).

## Overview

Category 062 carries **fused system tracks**. Obelix implements the full **User Application Profile (FRN 1–35)** from Edition 1.21. The Message Editor shows a UAP table and labels each form field with its **FRN** and **I062/xxx** item id.

## How to include an optional FRN

1. Open **Cat 062** in the Message Editor — the **UAP table** appears at the top.
2. Scroll to the matching form field (same **Form field** id as in the table).
3. For compound items: set **Include in FSPEC** to **Yes**.
4. For simple items: set a non-zero value (e.g. `service_id`, `flight_level`) or use the include toggle (`include_velocity`, `include_geometric_altitude`).
5. For complex variable items (FRN 11, 14, 16, 21, 22, 24, 27, 28, 34, 35): use **Raw payload hex** to paste bytes from Wireshark or an external tool.

## Mandatory items

Always encoded: **FRN 1, 4, 12, 13** (Data Source, Time, Track Number, Track Status).

## Limitations

- **FRN 2, 29–33** are spare — cannot be set (per Eurocontrol UAP).
- Variable-length items (1+) encode a **minimal valid item** (FX=0) unless **raw hex** is supplied. Full subfield editors for I062/380, I062/390, I062/340, etc. are not yet broken out into individual UI fields — use raw hex for production payloads.
- Track status encodes the **first extent only** (FX=0).
