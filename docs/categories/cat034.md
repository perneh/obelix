# Category 034 – Monoradar Service Messages

**Edition:** 1.29  
**Purpose:** Service and synchronisation messages from a monoradar station (not target plots).

## Overview

Category 034 carries **service messages** from a single radar source. Typical uses include north-marker pulses, sector-crossing indications, geographical filtering, and jamming strobe notifications. These messages help downstream systems align azimuth references and understand radar housekeeping events.

Obelix implements a focused subset suitable for testing decoders and simulating basic monoradar service traffic.

## Message types (I034/000)

| Value | Name | Azimuth (I034/020) |
|-------|------|--------------------|
| 1 | North marker message | Not included |
| 2 | Sector crossing message | **Required** |
| 3 | Geographical filtering message | Not included |
| 4 | Jamming strobe message | Not included |

When **Message Type** is set to *Sector crossing*, Obelix automatically adds **Antenna Azimuth (I034/020)** to the FSPEC and payload.

## Implemented data items

| Item | FRN | Obelix field | Encoding |
|------|-----|--------------|----------|
| I034/000 Message Type | 1 | `message_type` | 1 byte, values 1–4 |
| I034/010 Data Source Identifier | 2 | `data_source.sac`, `data_source.sic` | SAC + SIC, 1 byte each |
| I034/020 Antenna Azimuth | 3 | `azimuth` | 16-bit, LSB = 360°/2¹⁶ (only for type 2) |

## Example output

North marker (SAC=1, SIC=2):

```text
220007C0010102
```

Breakdown:

| Bytes | Meaning |
|-------|---------|
| `22` | Category 034 |
| `0007` | Data block length (7 bytes) |
| `C0` | FSPEC — FRN 1 and 2 present |
| `01` | Message type 1 (north marker) |
| `0102` | SAC=1, SIC=2 |

## Shared configuration

Load the preset **North Marker** from the library (`configurations/cat034/north-marker.json`).

## Wireshark

With the ASTERIX dissector enabled, filter on:

```text
asterix.category == 34 && udp.port == 8600
```

## Limitations

Obelix does **not** currently encode:

- Time-of-day items
- Additional UAP extensions beyond the three items above
- Message types 3 and 4 with any extra mandatory items from the full specification

For full UAP details, refer to the ASTERIX Category 034 specification (Edition 1.29).
