# Category 065 – SDPS Service Status Reports

**Edition:** 1.5 (Eurocontrol Part 15)  
**Purpose:** SDPS operational status, batch completion, and service event reporting.

## Overview

Category 065 is used by a **Surveillance Data Processing System (SDPS)** to report its configuration, health, and service-related events to downstream consumers. It complements Cat 062 system tracks and Cat 016 INCS configuration.

Typical use in Obelix Baltic scenarios: a static **SDPS Status** message at exercise start, before track/plot traffic.

## Implemented data items

| Item | FRN | Obelix field | Notes |
|------|-----|--------------|-------|
| I065/010 Data Source Identifier | 1 | `data_source` | Mandatory SAC/SIC |
| I065/000 Message Type | 2 | `message_type` | 1=SDPS Status, 2=End of Batch, 3=Service Status Report |
| I065/015 Service Identification | 3 | `service_id` | Omitted when 0 |
| I065/030 Time of Message | 4 | `time_of_message` | LSB = 1/128 s |
| I065/020 Batch Number | 5 | `batch_number` | Optional; auto-included for type 2 |
| I065/040 SDPS Configuration and Status | 6 | `sdps_configuration` | Included for message type 1 |
| I065/050 Service Status Report | 7 | `service_status_report` | Included for message type 3 |

### I065/040 — SDPS Configuration and Status

One-octet bit field:

| Field | Bits | Values |
|-------|------|--------|
| NOGO | 8/7 | 0=Operational, 1=Degraded, 2=Not connected, 3=Unknown |
| OVL | 6 | 0=Default, 1=Overload |
| TSV | 5 | 0=Default, 1=Invalid time source |
| PSS | 4/3 | 0=N/A, 1=SDPS-1, 2=SDPS-2, 3=SDPS-3 |
| STTN | 2 | Track re-numbering toggle |

### I065/050 — Service Status Report

Event codes 1–16 (service degradation, radar failures, contingency, synchronisation, etc.).

## Message types

- **Type 1 (SDPS Status):** Periodic operational heartbeat with I065/040.
- **Type 2 (End of Batch):** Marks completion of a processing batch; includes batch number.
- **Type 3 (Service Status Report):** Event-driven service notification with I065/050.
